# Coder Notes

This directory is repo-local operational memory for coding agents.

Use it to preserve:

- known-good runtime setups
- model-specific behaviors
- current bugs and regressions
- short dated implementation notes

Rules:

- keep entries concise
- prefer facts over narratives
- update or delete stale notes when the code changes
- do not duplicate AGENTS.md doctrine here

Files:

- `known-good.md`: validated setups and behaviors
- `known-issues.md`: current problems and sharp edges
- `coder-log.md`: short dated notes from real debugging and implementation work
