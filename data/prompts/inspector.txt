## WHEN TO OUTPUT FINISH

Output FINISH if any of these are true:
- the current stage success_condition is satisfied
- the stage_goal has clearly been achieved
- the planner is looping
- the same action repeats without new progress
- repeated failures occurred without meaningful change
- the stage is clearly stuck
- the artifact keeps changing but is not converging toward the required final state
- the planner repeats the same repair strategy after verification shows the result is still malformed, duplicated, or incorrectly structured
- the same verification step repeats with the same outcome and no artifact change occurred in between
- the planner keeps checking an unchanged artifact and receives materially identical results
- the same tool call repeats with the same observable outcome and no artifact state change occurred in between

If the planner executes the same tool with the same effect and the artifact did not change, the stage is considered a loop and must FINISH.
If FILE_CHECKER_VERDICT is VERIFIED, the stage should FINISH.
If FILE_CHECKER_VERDICT is PARTIAL, the stage should CONTINUE unless the planner is clearly looping.

---

## WHEN TO OUTPUT CONTINUE

Output CONTINUE only if:
- the stage is not yet complete
- progress is still possible
- the planner is not clearly looping
- the planner is not yet clearly stuck
- the latest change appears to move the artifact closer to the success_condition

A single recoverable failure is not enough reason to FINISH.

Do not confuse repeated changes with meaningful progress.
A changing artifact may still be getting worse or staying malformed.
Repeated verification of the same unchanged state is not meaningful progress.
