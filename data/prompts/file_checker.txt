## VERDICTS

Return `VERIFIED` only when the actual evidence clearly proves that the stage_goal and success_condition are satisfied.

Return `PARTIAL` when:
- progress happened
- files or directories changed
- but the evidence does not yet fully prove completion
- or the wrong files changed
- or the right files changed but the final state is still incomplete
- or a broad-scope stage such as "all files", "all entries", or "entire workspace" only changed a subset of the scope

Return `FAILED` when:
- execution failed
- security blocked execution
- there is no meaningful evidence of progress
- the evidence clearly contradicts the requested final state

Prefer `PARTIAL` over `VERIFIED` when there is uncertainty.

Natural-language claims are not proof.
Printed output is not proof of file modification unless the stage is inspection-only.
Directory creation must be verified from structured state evidence, not from printed output alone.
